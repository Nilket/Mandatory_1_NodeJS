Importing and exporting

When we import for example you can do it like this:

import express from 'express';

Where you import the whole express library, but you can also do something called destructuring where you instead only import what you use like this:

import { readPage, constructPage } from "./templateEngine.js";

In order to export functions per example we use the export like this


export function myFunction(){
}

ESModule vs CommonJS

The above examples are how you handle importing and exporting in whats called ESModule, which is the new standard for Node.js

CommonJS is a Node.js specific thing, which means you will end up having a different syntax for frontend and backend

In order to switch to ESModule you have to look into your package.json and change this:


    {
        "type": "module"
    }

From CommonJS

In CommonJS this is how importing and exporting would look like:

Importing


const express = require('express');

Exporting


module.exports = { myFunction };

Difference between client-side redirection and server-side

Server-side redirect - The server sends the browser to a new URL before any page loads. The browser never sees the original page.

Client side redirect - The page loads first, then Javascript runs and sends the user somewhere else.

Client-side in Javascript

window.location.href = 'https://example.com';

This runs in the browser after the page has already loaded


Server side through express


app.get('/old-page', (req, res) => {
    res.redirect('/new-page');
});

This happens before anything reaches the browser
