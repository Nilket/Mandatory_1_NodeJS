Semantic HTML

Semantic HTML is using elements that describe what the content is, rather than just how it looks

And it has it's pros

Accessibility - Vision impaired users can navigate the site

SEO - Google will understand the site much better if you use semantic HTML, and can even rank you higher because of this

Easier to maintain code - If all tags inside your HTML are divs it's gonne be much harder to navigate in and to understand

Pros and cons of CDN vs serving content locally

Serving scripts or styling through CDN (Content Delivery Network) vs locally each has it's pros and cons

Pros with CDN

CDN often have servers all over the world, this means that a user in Korea could access the files from a server in Asia insead of from a home server in Denmark, so it has faster load times for users all over the world

CDN puts less load on the server as we basically outsource all of the load to the CDN so the server has to do less work

CDN caches, and if the user has browsed other sites that uses some of the same scripts from the same CDN it's cached and won't have to be loaded again

Cons with CDN

Doesn't work offline

Dependent on the CDN

Pros with serving content locally

Works offline, if you're on the same network

Independent from the CDN so changes won't break your site.

Cons with serving content locally

Doesn't cache as CDN

Longer load times

SSR vs CSR

Where you render your HTML on a server or if it happens directly on a client's browser each has it's ups and downs

Client-side rendering

Scripts that are pubicly served are running as soon as the client receives the bare HTML

These scripts handle the HTTP GET request from the backend to gather the information and then render the content

Server side rendering

When a client calls an endpoint the backend does all of the rendering and only returns the HTML once all of the content is built.

It requires less of the client's computer as content is sent directly and doesn't need to be rendered
