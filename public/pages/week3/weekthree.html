Nodemon

Nodemon is a tool for Node.js that automatically reruns your application each time you make a change in the code

So developers don't have to spam this:

node app.js

But instead can type:

nodemon app.js

Now your application will rerun itself each time you update your code

To install nodemon you type this in your terminal

npm install -g nodemon

And you will be able to use nodemon.

Loop methods

In Javascript you can loop through items in a lot of different ways

But it's quite different from the ways we used to loop through things in Java, those manual counting for loops should only be for fingercounting problems.

But luckily for us Javascript has a lot of built in loop methods like these below:


.forEach()
const numbers = [1,2,3];
numbers.forEach((n) => console.log(n));

Which loops over all elements and runs a function per element, in this case 1,2,3.


.map()
const numbers = [1,2,3];
const multiply = numbers.map((n) => n * 2);

Which transforms each element and returns them to a new array, in this case 2,4,6


.filter()
const numbers = [1,2,3];
const overOne = numbers.filter((n) => n < 1);

Which returns a new array of all elements that pass for this case 2,3


.reduce()
const numbers = [1,2,3];
const sum = numbers.reduce((acc, n) => acc + n, 0);

Which accumulates all elements into a single value for this case 6.


.find()
const numbers = [1,2,3];
const found = numbers.find((n) => n > 1);

Returns the first element that passes the test in this case 2.


But you have to be aware because using these loop methods incorrectly can result in side-effects. Side-effects are common bugs that can lead to unexpected behaviour in the code, but they are also some of the hardest to debug.

Dangers of XSS

If your Javascript manipulates the DOM content using innerHTML based on your users input, there is a big risk of getting exploited by Cross-Site scripting


But the question is how do you avoid Cross-Site scripting?

To prevent it we sanitize all user inputs before injecting them into our DOM. Even better we can just use html elements textContent instead of innerHTML

For example:

element.textContent = userInput;
