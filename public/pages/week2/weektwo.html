Callback functions

In Javascript a callback function is simply a function that is passed as an argument to another function, which can then be called inside that function

Example:

function greetAnimal(animalName, makeSound){
    makeSound(animalName); //Callback with name passed into it
}

function meow(name){ 
    console.log(name + ' Says: Meow!');
} // The function that gets stored in makeSound

greetAnimal('Kitty', meow)

We have used callback functions a lot at the third semester but never knew the terminology

An example of where we used it:

button.addEventListener('click', function () {
    console.log('boom');
});

NPM

NPM is the built in package manager that Node.js comes with, it's what allows us to install dependencies for Node.js projects

To get started with NPM you can either create a package.json or generate one with:

npm init

After you run npm init, it will ask you a lot of questions that you have to fill out, that then gets filled into your package.json

What is package.json?

package.json is a file that describes your Node.js project in details

It contains meta-data for example the projects name, description, version. Most important it also contains a list of all the dependencies which are being used for the project, and even defined scripts.

An example of a package.json:


{
  "name": "mandatory_1_nodejs",
  "version": "1.0.0",
  "description": "A mandatory assignment",
  "keywords": [
    "Node",
    "EK"
  ],
  "homepage": "https://github.com/Nilket/Mandatory_1_NodeJS#readme",
  "bugs": {
    "url": "https://github.com/Nilket/Mandatory_1_NodeJS/issues"
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/Nilket/Mandatory_1_NodeJS.git"
  },
  "license": "MIT",
  "author": "Nilket",
  "type": "module",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "dependencies": {
    "express": "^5.2.1"
  }
}

To install these dependencies you simply use:

npm install

And it will install all the dependencies and put them into the node_modules folder, which you shouldn't push to git which i unfortunately did... woops


You can also install a single dependency at a time with:

npm install (dependency name)

Which automatically adds the dependency to package.json and installs it into node_modules


To run a script you can type:

npm run test

Which will run the script in the package.json

Express

Express.js is a light weight web framework for Node.js, you install it via npm. Express makes it easier to create and manage HTTP servers if we compare to creating your own

Express handles routing and request/response logic, which gets you started in no time.


In order to install express you use:

npm install express

Which adds it to your package.json and installs it into your node_modules folder

To use express you have to import it like this:

import express from 'express';

This imports the whole express library, but you have to create an instance in order to use it:

const app = express();

You also have to specify which port express needs to listen on in order for the webserver to start

const server = app.listen(PORT, () => {
    console.log("Server started on port", server.address().port);
});

Creating endpoints with express

To create an endpoint with express we use the instance of express like this:

app.get('/', (req, res) => {
        res.sendFile(path.resolve('public/pages/frontpage/frontpage.html'));
});

The endpoint can be made with whatever HTTP verb you choose.

Serving static files with express

In order to serve static files in Express.js you need to define your static/public folder like this:

app.use(express.static('public));

Which will make everything in this folder accessible to people using the server.
